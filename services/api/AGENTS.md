# services/api AGENTS

## Purpose

`services/api/` will contain the future FastAPI backend service.

## Responsibilities

Own HTTP boundaries, health checks, request validation, response shaping, configuration loading, and backend service composition.

## Constraints

Do not place domain rules, provider integrations, database-specific behavior, or browser automation directly in the API layer. Phase 0B.1 may document this boundary only; implementation waits for Phase 0B.2 approval.
