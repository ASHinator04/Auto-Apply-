# docs AGENTS

## Purpose

`docs/` contains supporting documentation that is too detailed or narrow for top-level files.

## Responsibilities

Use this directory for research notes, design sketches, phase summaries, glossary entries, and operational references.

## Constraints

Do not duplicate the canonical purpose of top-level documents. Link back to `PRD.md`, `ARCHITECTURE.md`, `ENGINEERING_GUIDE.md`, `ROADMAP.md`, `DECISIONS.md`, or `TODO.md` when those files own the topic.

## Extension Guidelines

Add subdirectories only when a documentation category has multiple files. Each substantial subdirectory should include its own `AGENTS.md`.
