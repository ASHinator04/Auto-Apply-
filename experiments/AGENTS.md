# experiments AGENTS

## Purpose

`experiments/` isolates research, prototypes, benchmarks, and exploratory work from production
architecture.

## Responsibilities

Use this area for LLM trials, embeddings research, prompt experiments, browser automation
exploration, and benchmarks.

## Constraints

Experiments must not become production dependencies by accident. Promote findings through
`DECISIONS.md`, specs, and contracts before implementation.

## Extension Guidelines

Each experiment should state its question, setup, result, and recommendation when it becomes more
than a scratch note.
